Почему нельзя применять команду git reset к коммитам которые выгружены на сервер Git?


1.Чтобы отказаться от изменений в файле (файлах) в рабочем каталоге, 
мы можем использовать команду git checkout -- <file> для восстановления 
состояния файла до последнего коммита. Если мы хотим отказаться от всех изменений 
во всех файлах рабочего каталога, можно выполнить команду git checkout -- . 
(ставим точку в конце команды).


2. Для удаления всех новых файлов, которые не должны быть включены в коммит, 
мы можем использовать команду git clean -f. Эта команда удалит все непроиндексированные 
файлы в рабочем каталоге. Если мы также хотим удалить папки, добавляем опцию -d, 
как git clean -f -d.

3. Чтобы вернуть файл из индекса в рабочий каталог, используем команду 
git reset HEAD <file>. Это удаляет файл из индекса, но оставляет его изменения
 в рабочем каталоге. Если мы хотим вернуть все файлы из индекса, выполняем git reset HEAD .
(точка в конце).

4.Если мы хотим отменить ошибочный коммит, есть несколько вариантов в зависимости 
от того, что вы хотите достичь:

-Если  мы хотим отменить последний коммит и оставить изменения 
в рабочем каталоге, используем команду git reset HEAD~ или git reset HEAD~1.
-Если мы хотим отменить последний коммит и удалить изменения, 
выполняем команду git reset --hard HEAD~ или git reset --hard HEAD~1.
-Если мы хотим отменить коммит, но сохранить изменения в рабочем 
каталоге, можно использовать команду git reset HEAD~ и затем git reset --soft HEAD@{1}.
-Если мы уже опубликовали (push) коммит в удаленный 
репозиторий, рекомендуется обсудить эту ситуацию с другими участниками 
проекта и рассмотреть возможности, такие как создание нового коммита, 
который исправляет ошибку.

Как исправить комментарии к коммиту?
Чтобы исправить комментарии к последнему коммиту, используем команду 
git commit --amend. Эта команда откроет текстовый редактор, где мы 
можем изменить комментарий к коммиту. Сохраняем изменения и закрываем 
редактор, чтобы завершить исправление комментария.

Как исправить отправленный коммит на GitHub?
Если мы уже отправили коммит на GitHub, рекомендуется быть осторожным 
при изменении истории коммитов. Если мы хотим исправить коммит, 
который уже отправлен на GitHub,  может потребоваться выполнить 
команду git push --force, чтобы перезаписать историю коммитов в 
удаленном репозитории. Однако, это может повлечь проблемы, если 
другие люди работают с репозиторием. В этом случае, рекомендуется 
обсудить ситуацию с другими участниками проекта и рассмотреть возможные 
последствия изменения истории коммитов.



Почему нельзя применять команду git reset к коммитам которые выгружены на сервер Git? 
Нельзя применять команду git reset к коммитам, которые уже были отправлены на 
сервер Git (например, GitHub), потому что команда git reset изменяет историю коммитов. 
Если мы изменим историю коммитов и отправим новую версию истории на сервер,
 это может вызвать проблемы у других людей, работающих с репозиторием.
